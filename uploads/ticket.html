📋 Objectif

Créer un Design System structuré et scalable pour le site YouSchool, avec une architecture tokens → composants professionnelle.

Contexte


Stack technique : WordPress + Elementor
Enjeux : SEO, conversion, réassurance, mobile-first

Utilisateurs du DS :
- Designer UX (création et maintenance)
- Intégrateurs (accès direct au fichier Figma)

🏗️ Architecture des variables Figma

Principe

Core (valeurs brutes) → Theme (alias sémantiques) → Composants
⚠️ Les valeurs ci-dessous sont indicatives. On les mets en V1 et on pourra ajuster facilement par la suite.

📁 Collection : Core

Palettes complètes, jamais utilisées directement.

Couleurs :

  • primary/50 → primary/900 (10 nuances)
  • secondary/50 → secondary/900 (10 nuances)
  • neutral/50 → neutral/900 (10 nuances)
  • success/500, success/600
  • warning/500, warning/600
  • error/500, error/600

📁 Collection : Theme

Alias sémantiques mappés aux Core tokens.

Texte :

  • text-primary → {core/neutral/900}
  • text-secondary → {core/neutral/600}
  • text-muted → {core/neutral/400}
  • text-inverse → #FFFFFF
  • text-link → {core/primary/500}
  • text-link-hover → {core/primary/700}

Backgrounds :

  • bg-page → {core/neutral/50}
  • bg-surface → #FFFFFF
  • bg-surface-alt → {core/neutral/100}
  • bg-overlay → rgba(0,0,0,0.5)

Interactive :

  • interactive-primary → {core/primary/500}
  • interactive-primary-hover → {core/primary/600}
  • interactive-secondary → {core/secondary/500}
  • interactive-secondary-hover → {core/secondary/600}

Borders :

  • border-default → {core/neutral/200}
  • border-strong → {core/neutral/300}
  • border-focus → {core/primary/500}

Status :

  • status-success → {core/success/500}
  • status-warning → {core/warning/500}
  • status-error → {core/error/500}

🧩 Composants à créer

Logique de construction : Atomic Design

Atoms → Molecules → Organisms → Sections → Templates

On construit les briques de base d’abord, puis on les assemble en composants plus complexes.

🔴 Priorité 1 — Atoms (briques de base)

Composants élémentaires, utilisés partout.

Boutons :

  • Button
    - Variants : Primary, Secondary, Outline, Ghost, Link
    - Tailles : sm, md, lg
    - États : default, hover, focus, disabled, loading
    - Avec/sans icône (left, right)

Champs de formulaire :

  • Input / TextField
    - États : default, focus, filled, error, disabled
    - Avec : label, placeholder, helper text, error message
    - Variante : avec icône (left/right)
  • Textarea
    • Mêmes états que Input
    • Resizable ou fixed
  • Select / Dropdown
    • États : default, open, focus, error, disabled
    • Avec : label, placeholder, helper text
  • Checkbox
    • États : unchecked, checked, indeterminate, disabled
    • Avec label
  • Radio Button
    • États : unselected, selected, disabled
    • Avec label
  • Toggle / Switch
    • États : off, on, disabled

Tags & Badges :

  • Badge
    - Variants : success, warning, error, info, neutral
    - Styles : filled, outline
    - Tailles : sm, md
  • Tag
    • Avec/sans icône close
    • Clickable ou static

Autres Atoms :

  • Icon
    - Set d’icônes utilisées (ou lien vers librairie)
  • Avatar
    • Tailles : sm, md, lg
    • Avec image ou initiales
  • Loader / Spinner
    • Tailles : sm, md, lg
  • Divider
    • Horizontal, vertical

🟠 Priorité 2 — Molecules (composants assemblés)

Combinaisons d’atoms pour créer des composants fonctionnels.

Formulaires :

  • Form Field (label + input + helper/error)
    - Standardise l’espacement et la structure
  • Form Group (plusieurs fields groupés)
  • Formulaire Lead
    • Champs : nom, email, téléphone, formation souhaitée
    • Versions : compact (sidebar) + complet (page)
    • Utilise les atoms Input, Select, Button

Cards :

  • Formation Card
    - Image, titre, durée, prix, badge CPF, CTA
    - Hover state
    - Versions : grid + list
  • Pricing Card
    • Prix, liste features, CTA
    • Variant “populaire” / mise en avant
    • Mention financement CPF
  • Testimonial Card
    • Photo, nom, formation, avis, note
    • Versions : compact + détaillée
  • Blog Card
    • Image, catégorie, titre, date, extrait

Feedback :

  • Alert / Notice
    - Variants : success, warning, error, info
    - Avec/sans icône, avec/sans close
  • Toast / Notification
    • Position : top-right, bottom-center…
    • Auto-dismiss
  • Tooltip
    • Positions : top, bottom, left, right

Navigation éléments :

  • Breadcrumb
  • Tabs
    • Horizontal, vertical
    • États : default, active, disabled
  • Pagination (⚠️ pas forcément nécéssaire dans notre cas pour l’instant)
  • Back to Top Button

Autres Molecules :

  • Search Bar (⚠️ pas forcément nécéssaire dans notre cas pour l’instant)
    - Input + bouton search
    - Avec suggestions (optionnel)
  • Rating / Stars
    • Note + étoiles + source (Google, Trustpilot)
  • Stats Counter
    • Chiffre + label
    • “+15 000 élèves”, “95% réussite”
  • Trust Badge
    • Icône/logo + texte
    • Certifications, labels qualité

🟡 Priorité 3 — Organisms (composants complexes)

Assemblages de molecules pour créer des blocs complets.

Navigation :

  • Header
    - Logo, menu, CTA
    - Version desktop (menu visible)
    - Version mobile (burger)
    - Version sticky
  • Mega Menu
    • Catégories de formations
    • Avec images/icônes par catégorie
  • Footer
    • Colonnes liens, réseaux sociaux, légal
    • Newsletter signup intégré
  • Mobile Menu (Drawer)
    • Menu burger ouvert
    • Navigation complète

Lead Generation :

  • CTA Sticky Mobile
    - Barre fixe en bas avec bouton principal
  • Pop-up / Modal Lead
    • Capture email (exit intent ou timer)
    • Avec formulaire intégré
  • Bandeau Promo
    • Annonces, offres limitées
    • Countdown timer optionnel
    • Dismissable (close button)

Content :

  • FAQ Accordion
    - Questions/réponses dépliables
    - Optimisé rich snippets SEO
  • Programme / Curriculum
    • Accordion avec modules
    • Icônes, durées par module
  • Table of Contents
    • Sommaire pages longues
    • Version sticky sidebar
  • Logo Wall
    • Grille logos partenaires / médias
  • Author Box
    • Photo, nom, bio courte (E-E-A-T)
  • Related Content Grid
    • Formations ou articles liés (3-4 cards)

Modals :

  • Modal générique
    - Header, content, footer (actions)
    - Tailles : sm, md, lg
    - Close button
  • Video Modal
    • Lightbox vidéo (témoignages, présentation)

Blocs de page prêts à l’emploi.
  • Hero Section

  • Features Section
    • Grille d’arguments (icône + titre + texte)
    • 3 ou 4 colonnes
  • Formations Grid
    • Grille de Formation Cards
    • Avec filtres optionnels
  • Pricing Section
    • 2-3 Pricing Cards côte à côte
    • Comparatif features
  • Testimonials Section
    • Carousel ou grille de témoignages
    • Avec stats de satisfaction
  • FAQ Section
    • FAQ Accordion + CTA contact
  • CTA Section
    • Bloc d’appel à l’action pleine largeur
    • Background coloré ou image
  • Stats Section
    • Chiffres clés en ligne
    • Counters animés
  • Trust Section
    • Logos + badges de confiance
  • Newsletter Section
    • Formulaire inscription newsletter

🔵 Priorité 5 — Templates de pages

Pages complètes assemblées avec les sections.
  • Home

♿ Accessibilité — Exigences obligatoires

⚠️ IMPORTANT : Le Design System doit respecter les normes d'accessibilité WCAG 2.1 niveau AA minimum.

Contrastes de couleurs

ÉlémentRatio minimumNorme
Texte standard (< 18px)4.5:1WCAG AA
Texte large (≥ 18px bold ou ≥ 24px)3:1WCAG AA
Éléments UI (icônes, bordures, focus)3:1WCAG AA
Texte sur images/dégradés4.5:1Vérifier le pire cas

Outils de vérification :

  • Plugin Figma : Stark ou Contrast
  • Site : WebAIM Contrast Checker

États interactifs

Tous les composants interactifs doivent avoir :

  • Focus visible — Outline ou ring clairement visible (pas juste un changement de couleur)
  • Hover distinct — Changement perceptible (pas uniquement couleur si daltonisme)
  • États disabled — Suffisamment distincts mais pas invisibles

Bonnes pratiques

  • Ne jamais utiliser la couleur seule pour transmettre une information (ajouter icône, texte, pattern)
  • Taille de texte minimum : 14px pour le body, 12px pour les captions
  • Touch targets minimum : 44x44px sur mobile (boutons, liens, inputs)
  • Espacement suffisant entre éléments cliquables

Checklist avant livraison

  • Tous les contrastes texte vérifiés avec Stark/Contrast
  • Focus states présents sur tous les éléments interactifs
  • Pas d'information transmise uniquement par la couleur
  • Touch targets ≥ 44px sur mobile
  • Testé avec simulation daltonisme (Stark)

✅ Règles de création

Structure obligatoire

  • Auto-layout sur TOUS les composants
  • Variables Figma utilisées (jamais de hex hardcodé)
  • Variants pour tous les états
  • Versions Desktop + Mobile quand pertinent
  • Naming clair : Component / Variant / State
  • Description dans le panel propriétés

Naming convention

Atoms : Button / Primary / Default
Molecules : Card / Formation / Hover
Organisms : Header / Desktop / Sticky
Sections : Section / Hero / Split

⚠️ À éviter

  • ❌ Valeurs hardcodées
  • ❌ Composants sans auto-layout
  • ❌ Composants sans variants d’états
  • ❌ Oublier les versions mobile
  • ❌ Naming générique (Frame 1, Group 2…)
  • ❌ Copier/coller au lieu d’instancier
  • ❌ Créer un composant composé sans avoir créé ses atoms d’abord
